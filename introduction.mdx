# Get Started with Finverse API

## **A. Developer authentication**

- Sign-up on our [Developer Portal](https://dashboard.finverse.com/) and obtain your API credentials (`customer_app_id`, `client_id` and `client_secret`).
  - **Data products**: you also need to register a valid `redirect_uri`.
- Input your API credentials (and `redirect_uri` for Data products) into Postman as Collection variables or Environment variables (e.g. go to Collection \> Edit \> Variables)
- Generate a `customer_token` to authenticate with Finverse's API (POST /auth/customer/token). Each `customer_token` expires after 60min, but can be re-generated. You should securely store the `customer_token` in your back-end to make other API calls.

## **B. Test vs. Live usage**

All API calls (both for test and live usage) should be made to our production environment: `https://api.prod.finverse.net/`

By default, when you create API credentials on our [Developer Portal](https://dashboard.finverse.com/), your app will be enabled as a "Test" app only. This allows you to make test payments (using test cards or our testbank) and access test data (by linking our testbank), but you can not make any real payments or access real banks.

- **Data products**: we can enable free live usage for testing purposes, subject to a limited quota for your app. Please contact [support@finverse.com](https://mailto:support@finverse.com/) to request this

To enable live usage:

- Go to our [Developer Portal](https://dashboard.finverse.com/) and setup a new team (to separate live vs. test usage)
- Go to API Settings to create API credentials for your new "Live" team
- Contact [support@finverse.com](https://mailto:support@finverse.com/) and specify your "Live" team's Customer App ID (viewable under API settings). Finverse will then turn on live usage for your app (this typically requires signing commercial documents which Finverse will send to you)

# **Getting started with the Finverse Link UI**

The Finverse Link UI provides a pre-built UI allowing end-users to use the Finverse platform without any additional UI work from you. In particular, with the Finverse Link UI, an end-user can:

- **(Payments products)** Make a payment, setup a payment method (using Payment Links), or setup direct debit mandates (using Mandates)
- **(Data products)** Select and link their bank, to access & share their bank data

## **Pre-requisites**

First generate a Link `url` or `link_url` using the following end-points:

- **Payments products:**
  - **Payment Links** (recommended integration): `POST /payment_links`
  - **Direct Debit Mandates:** `POST /mandates/link`
- **Data products**: `POST /link/token`

Refer to the documentation on each of the above endpoints for details.

## **Platform-specific considerations**

### **Finverse Link UI in a Web App**

When opening the Finverse Link UI in a Web App, it is important to note that we currently support the following `ui_mode`:

| `ui_mode` value | Description |
| --- | --- |
| iframe | Default mode. In this mode, Finverse Link UI expects the `link_url` to be opened in an iframe. The parent app is responsible for closing the Finverse Link UI (e.g. after a success or failure screen, or when the user clicks a close/exit button). |
| redirect | In this mode, the Finverse Link UI expects the `link_url` to be directly opened in a browser. When a user clicks a button to close/exit/finish the flow, the Finverse Link UI will redirect the user to the `redirect_uri` provided when the Link `access_token` was created. |
| auto\_redirect | Same as `redirect` mode, but in this case the user will be automatically redirected if the user doesn't perform any action on the success or failure screen. |
| standalone | In this mode, the Finverse Link UI expects the `link_url` to be directly opened in a browser, without any UI redirect at the end of the Link flow. The Finverse Link UI will prompt the user to close the browser tab upon completion of the flow. |

#### **Closing iframe from the parent web app**

When in `iframe` mode the Customer App (parent app) will open Finverse Link in an iframe like the following code.\
**Code**

When the user presses the continue or the close button after linking, it will send a message back (either `close` or `success`) to the parent app. The Customer App will need to handle it as follow:

```javascript
// add a listener so we get the message sent by the iframe
window.addEventListener('message', receiveMessage, false);
const receiveMessage = (message: any) => {
    if (message.data === 'close') {
        // do whatever is required to close the iframe. The linking did not succeeded due to user pressing the close button.
        setLinkOpen(false);
    }
    if (message.data === 'success') {
        // do whatever is required to close the iframe
        setLinkOpen(false);
        // Since the linking was successful display the next screen
        history.push('/user');
    }
};
```

## **Product-specific considerations**

### **Finverse Link UI for Payments**

When using the Finverse Link UI to authorize Payment Links or Mandates, a webhook will be called when the Payment Link / Mandate is successful (or fails). Contact the Finverse team to configure this `payment_webhook`.

### **Finverse Link UI for Data retrieval**

When using Finverse Link to retrieve data it will eventually redirect to the `redirect_uri` supplied when creating the link token. It will pass along `code` and `state`. The `code` should be exchanged for a Login Identity `access token` (see `POST /auth/token`) and the `state` can be used to reconcile which user the token should be associated with.
